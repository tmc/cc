Recommendation

Use ansi_up for Claude Code session replay.

It's lighter (15KB vs 300KB), simpler to implement, and better suited for structured JSONL session data. Reserve xterm.js only if you need full terminal emulation with cursor positioning.

Key Research Findings

✓ xterm.js can render without a backend - Use disableStdin: true for read-only mode

✓ Available via esm.sh CDN - No build step required, works in browser

✓ Addons available - FitAddon (essential), SearchAddon, WebglAddon

✓ ansi_up is simpler - Better for structured Claude Code session data

✓ SSE integration works great - Stream JSONL events from Go backend
