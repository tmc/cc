# Claude Code Utilities (cc) - Project Instructions

This is a collection of core utilities for developing software with Claude Code.

## Project Overview

The `cc` package provides utilities for:
- **cmsg**: Format stdin as Claude messages (NDJSON/streaming compatible)
- **creplay**: Replay Claude Code sessions in a TUI (supports streaming/realtime)
- **mksid**: Generate timestamp-sorted UUIDs with git repo context for session tracking

## Development Guidelines

### Code Style
- Follow Russ Cox's coding style (simple, clear, minimal)
- Keep utilities focused and composable
- Prioritize streaming/real-time compatibility where applicable
- Use standard Go idioms and patterns

### Documentation
- All commands MUST have doc.go files following https://go.dev/doc/comment
- Package documentation should clearly state the purpose and use case
- Command documentation should explain input/output behavior and streaming support

### Session ID Format
- Session IDs are generated by `mksid`
- Format: timestamp-sorted UUID + git repo hash + fork tracking capability
- Use these IDs consistently across all cc utilities for session correlation

### Streaming and NDJSON
- Tools should support streaming input/output where practical
- Use NDJSON format for structured data interchange
- Ensure compatibility with Unix pipes and standard streams

### Testing
- Test with real Claude Code session data where applicable
- Verify streaming behavior doesn't buffer unnecessarily
- Test git repo detection across different directory structures

### Git Workflow
- Stage changes atomically (separate logical changes)
- Use ~/bin/git-auto-commit-message --auto for commit messages
- Add git notes with session metadata after commits
- Never stage binary files

## Tool-Specific Notes

### cmsg
- Default message type: user
- Must preserve streaming semantics
- Handle stdin gracefully (EOF, partial reads)

### creplay
- TUI should handle both historical and real-time playback
- Support session ID lookup and validation
- Gracefully handle incomplete/corrupted sessions

### mksid
- Include CWD git repo hash for context correlation
- Reserve space in UUID format for fork tracking
- Must be timestamp-sortable for chronological ordering

## Integration Points

These utilities are designed to work together:
```bash
# Example: Create session, format message, replay later
export SID=$(mksid)
echo "user input" | cmsg | tee session-$SID.ndjson
creplay $SID
```

## Build and Install

```bash
# Build all commands
go build ./cmd/...

# Install to GOPATH/bin
go install ./cmd/...
```
